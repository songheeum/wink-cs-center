# 단비공감팀 상담가이드 - 정적 대시보드 초안

## 파일 구조

```text
index.html  : 화면 뼈대
style.css   : 라이트/다크 테마 및 레이아웃 스타일
app.js      : CSV 로드, 집계, 화면 렌더링 로직
app.json    : 앱 설정, CSV 주소, 메뉴/카드 데이터, 보안 제외 컬럼 설정
```

## GitHub Pages 업로드 방법

위 4개 파일을 저장소 루트에 업로드하면 됩니다.
`index.html`이 `style.css`, `app.js`, `app.json`을 같은 폴더에서 읽습니다.

## 데이터 보안 기준

화면/검색/상세 테이블에서 아래 컬럼은 제외합니다.

- D열: 자마드 주소
- F열: 메모
- M열: 특이사항

주의: 현재 방식은 브라우저에서 공개 CSV를 불러와 화면에서 숨기는 방식입니다. 원본 CSV 자체에 민감정보가 들어 있으면 CSV 링크를 가진 사람은 원본을 볼 수 있습니다. 완전한 보호가 필요하면 D/F/M열을 제거한 공개용 시트를 따로 만들어 `app.json`의 `csvUrl`에 연결해야 합니다.

## 대시보드 기준

- 월별 통계: 접수일 기준
- 완료: 상태값에 `처리완료`가 포함된 건
- 취소: 상태값에 `취소`가 포함된 건
- 기타: 완료/취소 외 상태
- 소요시간: 시간 기록이 있는 건 기준이며, 교사용 업무폰 및 PC 점검 건 중심의 투입시간으로 표기
