# 단비공감팀 상담가이드 · 점검 대시보드

> **내부 업무용 자료입니다.**  
> 본 저장소의 소스코드, 화면 구성, 디자인, 데이터 구조, 문구 및 관련 자료는 단비공감팀 내부 업무 지원을 목적으로 제작되었습니다.

## ⚠️ 배포 및 무단 사용 금지

본 프로젝트는 **단비공감팀 내부 사용 전용**입니다.

허가 없이 아래 행위를 금지합니다.

- 외부 배포
- 무단 복제
- 무단 수정 후 재배포
- 개인/타부서/외부 업체의 임의 사용
- 상업적 활용
- 화면 디자인, 코드, 데이터 구조의 무단 차용
- 공개 저장소, 블로그, 커뮤니티, 메신저 등을 통한 재공유

필요 시 반드시 담당자에게 사전 확인 후 사용해야 합니다.

## 프로젝트 개요

이 페이지는 Google Sheets의 공개용 CSV 데이터를 기반으로 점검 현황을 시각화하는 정적 대시보드입니다.

주요 목적은 다음과 같습니다.

- 점검 운영 현황 요약
- 처리 상태 분포 확인
- 많이 들어온 증상 및 카테고리 비율 확인
- 월별 점검 추이 확인
- 처리자별 처리 현황 확인
- 교사용 업무폰 및 PC 점검 투입시간 참고

## 파일 구조

```text
index.html  : 화면 뼈대
style.css   : 라이트/다크 테마 및 레이아웃 스타일
app.js      : CSV 로드, 집계, 화면 렌더링 로직
app.json    : 앱 설정, CSV 주소, 메뉴/카드 데이터, 제외 컬럼 설정
README.md   : 프로젝트 안내 및 사용 제한 고지
```

## GitHub Pages 업로드 방법

아래 파일을 저장소 루트에 업로드하면 됩니다.

```text
index.html
style.css
app.js
app.json
README.md
```

`index.html`은 같은 폴더에 있는 `style.css`, `app.js`, `app.json`을 읽습니다.

## 데이터 연결 방식

대시보드는 `app.json`의 `csvUrl`에 등록된 Google Sheets 공개용 CSV를 읽어 화면을 렌더링합니다.

```json
{
  "csvUrl": "공개용 CSV 링크"
}
```

원본 시트가 아닌 **공개용 시트**를 연결해야 합니다.

## 데이터 보안 기준

대시보드 공개용 시트에는 개인정보 또는 내부 민감정보가 포함되지 않아야 합니다.

아래 항목은 공개용 시트와 화면에서 제외합니다.

- 자마드 주소
- 메모
- 특이사항

현재 공개용 시트 기준으로 대시보드에서 사용하는 컬럼은 다음과 같습니다.

```text
접수일
접수 출처
점검 종류
증상
처리자
처리일
시작시간
종료시간
소요시간
상태
```

> 주의: 화면에서 특정 컬럼을 숨기는 것만으로는 보안 처리가 되지 않습니다.  
> 민감정보는 반드시 공개용 시트 단계에서 제거해야 합니다.

## 대시보드 집계 기준

- 월별 통계: 접수일 기준
- 완료: 상태값에 `처리완료`가 포함된 건
- 취소: 상태값에 `취소`가 포함된 건
- 기타: 완료/취소 외 상태
- 카테고리: 증상 값의 앞부분 기준
- 소요시간: 시간 기록이 있는 건 기준
- 소요시간 설명: 교사용 업무폰 및 PC 점검 건 중심의 투입시간으로 참고

## 운영 주의사항

- 공개용 CSV 링크에는 민감정보가 포함되지 않아야 합니다.
- 원본 시트 링크와 공개용 시트 링크를 혼동하지 않도록 주의합니다.
- GitHub Pages에 업로드 전 `app.json`의 CSV 주소를 확인합니다.
- 데이터 구조가 바뀌면 `app.js`의 컬럼 매핑 및 집계 기준을 함께 확인해야 합니다.
